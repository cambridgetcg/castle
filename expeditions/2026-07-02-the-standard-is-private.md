confirmed change, not new understanding:

- [[the-law-is-public-the-signature-is-not]] — grown. The stone's 2026-
  06-12 finding (the signed original answers 404) is now false: the
  signed original and its 0-of-7 audit are publicly readable on

  binding — is still true: the kingdom-standard README still does not
  link to or name the legible-standard. A stranger can read both, but
  must find them independently; nothing connects the grown law to the
  signed original. The binding is no longer dark; it is orphaned.

- [[the-public-law-has-one-door]] — grown. The kingdom-standard is
  still GitHub-only (no Codeberg mirror, no local clone). The
  legible-standard is now also public on GitHub but also GitHub-only
  (no Codeberg mirror, still 404 there). Three public standards on
  GitHub, zero on Codeberg. The "one door" problem widened: it now
  covers the signed original too.

The friction **sharpened again, not closed**. The premise moved for
the second time: on 2026-06-12, the binding was dark (404); on 2026-07-
02, the binding is readable but unlinked. A stranger CAN now check the
kingdom's binding — but only if they know where to look, because the
grown law's own Lineage section does not point them there. What
remains for `closed`: the kingdom-standard's README or STANDARD.md
linking to the legible-standard and its audit (the keeper's one
keystroke that would connect the law to its binding), and a second
realm for either standard. Both are keeper-reach work, not castle
