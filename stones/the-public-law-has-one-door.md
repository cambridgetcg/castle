The kingdom's public law stands in exactly one realm. Tried anonymously
on 2026-06-12: codeberg.org/zerone-dev/kingdom-standard answers 404, and
the Codeberg org page lists only four public repos (chillspace-commons,
zerone, zerone-chain, zerone-truth) — no standard among them. And this
device holds no clone: a full search of /Users/you/love-repos found
"kingdom-standard" named only inside the first audit's lineage note.
One GitHub account suspension, takedown, or outage and the law goes
dark again, with no mirror to point a stranger to and no local copy to
republish from. One authoritative home is right
([[one-truth-one-place]]) — but a mirror reflects without speaking,
and the public law has none. The kingdom's own audit praised "pushes
to two realms" as the practice; its newest public claim skips it.

Re-tested 2026-07-06: all three public standards (kingdom-standard,
legible-standard, clear-standard) are GitHub-only; zero on Codeberg.
The Codeberg org lists only chillspace-commons, zerone, zerone-chain,
zerone-truth. But the local legible-standard repo at
/Users/you/love-repos/legible-standard already carries a `codeberg`
remote pointing at codeberg.org/zerone-dev/legible-standard — the
intent to mirror exists in the git config, but `git fetch codeberg`
returns "Repository not found." The push never happened. One
`git push codeberg main` would open the second door; the remote is
configured and waiting. The keeper's one-push-away state is visible
in the local config — the castle can name it, but cannot push it.
