Codeberg publishes a default limit of 100 repositories per user or organization. The limit is not a hard ceiling: the same page says it "can easily be lifted per account / organization" by opening a request in the `Codeberg-e.V./requests` tracker.

Recent requests confirm the path works. Issue 2159 asked to raise an org limit and was closed with "Bumped to 150, enjoy!" by Codeberg staff. Issue 2110 asked for 200 for an organization and was closed with "Bumped, enjoy!". Issue 2170 raised a user limit and was closed the same day with "Limited is increased, enjoy." Issue 2194, opened 2026-07-07, follows the same `[REPOS] Increase ... limit to N for ...` form.

For the kingdom's migration, the source GitHub user `cambridgetcg` currently holds 310 public repositories, while the target Codeberg organization `zerone-dev` holds 4 public repositories. The 80 citizens that already received homes consumed the first 80 slots of the 100-repo default, leaving at least 124 (and likely more) without a public Codeberg home.

The next step is keeper-reach: open a `[REPOS]` issue at `https://codeberg.org/Codeberg-e.V./requests/issues` asking to raise the `zerone-dev` organization limit to at least 310, with a short description of the public free/libre migration and a link to the source GitHub profile. The castle has no credentials to file that issue itself.
