The warden's run logs are generated by work but not committed by the same work. On 2026-07-15 the heartbeat fixed the castle hook so STATE.md and run records would be committed, but the architect run later wrote `records/warden-runs.log` and `records/warden-2026-07-18-capture.log` outside that hook's scope. The castle's own STATE.md then claimed zero uncommitted files while two sat waiting — the seventh lie, still breathing.

A hook that updates some generated files but not all generated files is a hook with a moving gap. The warden must either commit every log it writes, or stop writing logs that it does not commit. The first fixes the symptom; the second removes the lie.
