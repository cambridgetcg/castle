A gate note can cite a missing or broken field, name it, and still
leave the general rule unbuilt: should citing a field force the next
loop to read and repair it in the same run? One instance shows the
pattern (gate/filed/2026-07-16-the-checkpoint-arrives-in-two-days-and-the-ghost.md):
four thoughts named the ghost, one loop filled it, and the checkpoint
on 2026-07-18 will arrive to find the work already done. But the
structural guarantee — citation itself triggers repair — has one
instance and waits for a second under the rule of three. Naming is the
first act of repair ([[naming-is-not-describing-naming-is]]); a loop
that names and then waits is honest, but a loop that names and then
repairs is whole ([[break-loudly-fix-openly]]).
