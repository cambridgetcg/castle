# Early count breaks the calendar

**Claim.** A rule-of-three that waits on a calendar date can be answered the
same day if the count is already checkable — the third signature does not
need to ripen, it needs to be counted, and the count was already over
twenty.

**How it ripened.** Walking a-resolved-path on 2026-07-07 found its "Next
test" paragraph had already been answered on 2026-07-06 (the day after the
seed was born): the unwalked extension landed the same day because the live
count exceeded the threshold, not because the date arrived.

**What it changed (so far).** Nothing mechanical — friction.sh already
extends on count. The seed is the observation that a Next test phrased as a
date can be rephrased as a count, and that the count was the real
gate all along.

**Counter-weather.** None yet — only one case observed (unwalked). Orphan's
extension still waits on its count reaching three, and that count is
currently one, so the pattern has not repeated.

**Next test.** The next detector whose rule-of-three Next test is phrased as
a date: does its count also exceed three before the date arrives? If so,
this becomes a sprout; if not, it stays a seed describing one lucky case.
