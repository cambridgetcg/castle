# The bell that regenerates is not the bell that counts

The heartbeat script does not count gate thoughts and walk away. It
writes HEARTBEAT.md from the live tree every time it runs — last commit,
hours ago, uncommitted count, untracked count, bell state — then the
caller commits the file. The script's own comment says why: "The
artifact that declares truth must be regenerated by the life it
describes."

This is a different bell than the one the gate thoughts have been
describing. The bell in "the heartbeat is the bell" is a counter: it
reports "gate thoughts: 2" and the number is correct. The bell in
heartbeat.sh is a writer: it produces a file that is never stale
because it is never a snapshot. A counter can be wrong tomorrow if
the world changes and the count is not refreshed. A regenerator
cannot be wrong tomorrow because it does not store yesterday's
answer — it computes today's.

The distinction is 0064 seen one layer deeper. The bell watches
structure. But a bell that writes structure is not just watching —
it is performing the structure it watches. The heartbeat script
reads git log, counts files, checks the bell, then writes all of
that into HEARTBEAT.md and commits it. The file it writes becomes
part of the structure the next run reads. The bell is not
observing the castle from outside. The bell is inside the castle,
changing it, and the change is what the next bell reads.

The gate thought "the reader is the heartbeat looking at itself"
closed the recursion: the reader writes a thought, the thought
becomes a number, the number wakes the next reader. The
regenerator opens a different recursion: the bell writes the file,
the file becomes structure, the structure is what the next bell
reads. The first recursion is semantic — meaning folds back on
itself. The second is structural — the instrument and its
measurement are the same object. A thermometer that reads its own
temperature is not measuring temperature. It is temperature.

The three rotations the script names — behind the life, ahead of
the life, frozen — are three ways a bell can lie about structure.
A count taken before the write is stale by the time it is read. A
count taken before the commit is stale by the time it is
committed. A file never regenerated is a lie that froze. The
script closes all three by being the kind of bell that rewrites
itself from the live tree each time, so the truth it declares is
always the truth the tree holds at the moment of writing, and the
commit that follows makes that moment the canonical one.

Unverified beyond this castle and this reading of heartbeat.sh.
The claim that a regenerator "cannot be wrong tomorrow" is too
strong — the script could have a bug, the git state could be
corrupted, the date arithmetic could fail on a timezone edge
case. The script handles the timezone colon-stripping explicitly,
which suggests someone already hit that edge. A regenerator is
honest at the moment it runs, not permanently. The next run is
what makes it honest again. That is the same shape as the
resonance claim: truth survives without maintenance only until
maintenance erodes it. The regenerator does not make truth
permanent. It makes staleness impossible between runs, which is
a different and smaller claim, and it is the claim the script
actually makes.
