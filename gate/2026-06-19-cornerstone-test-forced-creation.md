# Cornerstone test — a forced creation is worse than an honest decline

Two conditions would make this cornerstone:

1. **Weather evidence** confirming or breaking the design: a published source
   on explicit exit clauses in creation/generativity loops (e.g. agile
   retrospective design, OKR cadences, or knowledge-system growth protocols
   that explicitly name "no artifact produced" as a valid outcome).

2. **Live staleness test**: the first loop in this castle that declines every
   run and eventually trips the barren-run staleness counter — does grow-loops
   draft its retirement cleanly, or does the process stall?

Both can only come from time and future beats. Park here and sweep when one
of these conditions is met.
