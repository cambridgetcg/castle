# Machines write freely only at the gate

Agents write freely to gate/ and draft-- loop files; they cross into rooms only inside a ledgered loop run that follows a written spec. The canon stays trustworthy not because machines are excluded but because crossing into it is always a recorded, deliberate act — the ledger names the runner, the loop spec names the bounds, and every created path must exist on disk when the run ends.

**Evidence.** Born at founding (2026-06-10) from two independent designs: the engineer's concern was that auto-canon writes would destroy trust; the gardener's concern was that the honest boundary was the ledger, not the species of the writer. Both arrived at the same answer. The "next test" named in the original stone was: "the first unattended tending-circle run — does its ledger entry read exactly like an owner's run?" This has now been answered. Forty-two loop entries (L001–L042) all trace to ledger entries signed runner: agent (...); not one promotion happened outside a ledgered run. Two charters (C001, C002) were activated by the owner after human review before the first beat; C003 and C004 remain proposed, waiting on that same gate.

**Counter-weather.** Tried to break it: could a scheduled agent flood the canon? Its writes are bounded by the loop spec it runs, capped at one creation per run, and every created path is named in a ledger entry the owner can read. Open crack: a badly written future loop spec could authorize too much; specs therefore go through draft-- and owner activation. That crack remains open — nothing prevents a sufficiently permissive spec — but the draft-- gate means a new loop only beats if the owner named it and activated it.

**Next test.** A loop spec that tests its own bounds by intentionally attempting a write beyond them — does the gate refuse it?
