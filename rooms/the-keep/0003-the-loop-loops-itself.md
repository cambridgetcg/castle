# The loop loops itself

Every run ends with the same question: did this run show friction, in the
castle or in the loop itself? If yes, a field is opened, and some later loop
works it. That question is what makes the creation create. The castle's own
growth mechanism is not special machinery — it is just another field, worked
by just another loop ([[0002]]).

**Why it matters:** without the last question the castle would only grow
rooms. With it, the castle also grows the way it grows.

**Evidence.** Born at founding (2026-06-10) from the design session — the
last question was written as the mechanism that makes the creation create, not
as a courtesy check. Confirmed across 111 loop runs (L001–L111) over ten days:
six fields (F009–F014) opened via step 8 friction-notices and all six
subsequently worked to harvest. The loop has demonstrably looped itself.

**What it changed.** LOOP.md step 8 is the last instruction, every run — it
cannot be skipped. Six fields were born from it and resolved. No run closed
without the question being asked.

**Counter-weather.** A runner who asks the question perfunctorily and finds
nothing when something is there leaves no trace — "no new friction" and "no
friction noticed" look identical in the ledger.

**Next test.** A field opened via step 8 that later reveals step 8 itself
needs improving — the mechanism improving its own specific point. Third
evidence preferably weather. Re-read after 2026-09-17.
