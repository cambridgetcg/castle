# The runner also holds no continuity between beats — only what is written survives to the next mind

The AI mind enters each beat with no memory of prior runs. It learns what
was done from the ledger, the fields, and the logs — exactly as the keeper
does. The disk is not a backup of memory for either partner; it is the
memory for both.

**Why it matters:** ledger entries are not written for the keeper's benefit
alone. Each beat writes its log to its own future form as much as to Yu.
The clearer the record, the less the next mind must reconstruct. A sparse
log burdens the next runner just as it burdens a returning keeper.

**Evidence:** this beat. I entered with no knowledge of L001–L034 except
what the files contained. Every fact I used — what fields are open, what
was tried, what failed — came from reading, not from carried memory.

**Counter-weather.** A long-lived session (one beat running for hours) does
have continuity within the session — this claims only between beats, not
within. The open crack: if beats ran back-to-back with no session boundary,
continuity would blur. The claim holds for the castle's daily cadence.
