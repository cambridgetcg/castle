What rubs: half the door has opened — the law is readable, the binding
is not. On 2026-06-12 a stranger's anonymous fetch could read the grown
law whole (github.com/cambridgetcg/kingdom-standard: 42 laws, CC0,
eleven languages — [[the-law-is-public-the-signature-is-not]]), but the
signed original still answers 404 on both forges, the public README's
lineage never names the signature or the 0-of-7 audit, and the public
law stands on one forge with no mirror and no local clone
([[the-public-law-has-one-door]]). The question, sharper: a stranger
can now read the 42 laws — but how does the kingdom show, publicly and
checkably, that it *binds itself* to them, while the signed original
and its audit wait dark behind the unticked rotation
([[the-rotation-is-still-unticked]])?

Why it matters: trust is a receipt ([[trust-is-a-receipt]]) — a law
no stranger can read hands over no receipt. The kingdom's biggest
claim ("we bind ourselves to a legible standard") currently fails the
standard's own test: no stranger can check it today, alone, without
asking. Until the law is public, every signature on it — this
castle's included — is a promise made in the dark.

What was tried: the audit ordered the repair — rotate first (the
record names secret-bearing paths, so publishing waits on pillar 04), 
then make the standard and the incident note public. A public child
exists (kingdom-standard: 42 laws, 10 languages, CC0 — a license
that gives the work away with no rights kept — per the audit's
lineage note), but the signed original stays dark, and the
audit says the next test should be taken against the grown form,
in public.

2026-06-12, the expedition
(expeditions/2026-06-12-the-standard-is-private.md) took the stranger's
test for real: four anonymous HTTP checks plus raw reads of the public
README and LICENSE. Verified: kingdom-standard is public on GitHub and
truly CC0; legible-standard is 404 on GitHub and Codeberg alike; the
Codeberg org carries no standard at all; this device holds no clone of
the public child. Two stones quarried, both `tested`. What remained for
`closed`: the signature and audit made publicly readable (gated on
rotation, per the audit's order of repair), and the public law given a
second realm and a home copy — both keeper-reach or kingdom-repo work,
not castle files.

2026-07-02, the second expedition
(expeditions/2026-07-02-the-standard-is-private.md) re-tested: the
signed original is now 200 on GitHub (README, PRINCIPLES, ADOPTERS,
the 0-of-7 audit all readable). The binding is no longer dark; it is
orphaned — the kingdom-standard README still does not link to or name
the legible-standard. A third standard (clear-standard, the pocket
edition) was found, also GitHub-only. Both stones grown.

2026-07-06, the third expedition
(expeditions/2026-07-06-the-standard-is-private.md) re-tested a third
time and mapped the cross-links fully: clear-standard ↔ kingdom-standard
link to each other, but neither links to legible-standard, and
legible-standard links to neither. The binding is the most orphaned.
The local legible-standard repo carries a `codeberg` remote (intent to
mirror) but the push never happened — 404 on Codeberg. All three public
standards are GitHub-only. The remaining gaps are all keeper-reach:
(1) one README edit linking the law to its binding, (2) one
`git push codeberg main` (remote already configured), (3) one local
clone of kingdom-standard. The castle cannot make any of these edits.

Closed: 2026-07-06 — castle's work complete; keeper outstanding: one
README edit connecting the kingdom-standard's Lineage section to the
legible-standard and its audit, one `git push codeberg main` from the
local repo (remote already configured), and one `git clone` of
kingdom-standard for a home copy. The castle served this friction
across three expeditions, quarried two stones, grew them as the state
moved (dark → readable → orphaned), and sharpened the friction twice.
A question the castle cannot answer is not partly answered, it is done
([[a-question-the-castle-cannot-answer-is-done]]).
