# chronicle: castle mend — 2026-06-10

*Written by the publisher applying an outside reviewer's findings — the same
hand as `records/2026-06-10-custodian.md` and `records/2026-06-10-mend.md`,
where the full account and reasons live. Old and new wording stand here in
full, as the castle loop's protocol demands, so every change can be checked
and undone by a later run if it proves wrong.*

## the rub

The custodian ruled that every loop checks `HALT` and `STOP` before acting —
yet none of the five protocols in this design contained any such check, and
the warden's first standing order was invisible to the tools because its
friction file had no frontmatter.

## change 1 — the kill-switch line, added to all five protocols

In `loops/masonry.md`, `loops/architecture.md`, `loops/friction.md`,
`loops/keep.md`, and `loops/castle.md`, the opening stanza read:

> Work from the castle root — the folder that holds this `loops/` folder.
> Today's date in YYYY-MM-DD form: `date +%F`.

It now reads, in each:

> Work from the castle root — the folder that holds this `loops/` folder.
> Today's date in YYYY-MM-DD form: `date +%F`.
> Before anything: if a file named `STOP` or `HALT` exists in the castle
> root, stop silently — do nothing and write nothing.

One line, the same line, five doors. Nothing else in any protocol changed.

## change 2 — the standing order made visible

`frictions/2026-06-10-two-vocabularies-one-castle.md` lacked the friction
frontmatter, so `./castle.sh` counted 4 open frictions where 5 existed and
this loop's own trigger (`room: castle`, `status: open`) could not see it.
Added above the existing text, deleting nothing:

> ---
> name: two vocabularies, one castle
> born: 2026-06-10
> status: open
> room: castle
> ---

and the heading `## What has been tried` became the schema's
`## What was tried`. The tool now counts 5; the trigger now fires.

## why one entry holds two changes

Both came from one reviewer's walk and one publisher's hand, applied together
before publication; splitting them across invented entries would order the
chronicle by fiction. The records on the other side of the root hold the
remaining mends to that design's own contracts.
